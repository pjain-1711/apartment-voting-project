PROJECT OVERVIEW:
-----------------
A complete, production-ready electronic voting system for apartment complex
wing representative elections. Built with Flask, SQLite/PostgreSQL, and
Bootstrap 5.

WHAT WAS BUILT:
--------------
✓ Wing-based voting system with duplicate prevention
✓ Admin panel with secure authentication (bcrypt)
✓ Wing and nominee management (CRUD operations)
✓ Real-time voting progress tracking
✓ Automated result calculation and winner declaration
✓ Excel exports (3 types: anonymous, detailed, results)
✓ Configurable settings (voting on/off, winner count, results visibility)
✓ Mobile-responsive Bootstrap UI
✓ Security features (CSRF, rate limiting, SQL injection prevention)
✓ Edge case handling (0/1/2+ nominees scenarios)
✓ Sequential vote counter system
✓ Full documentation (README + QUICKSTART guide)
✓ Git repository initialized

GETTING STARTED:
---------------
1. Install dependencies:
   cd voting-system
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt

2. Run the application:
   python run.py

3. Access at: http://localhost:5000

4. Admin login:
   URL: http://localhost:5000/admin/login
   Username: admin
   Password: admin123
   ⚠️ Change these in production!

EDGE CASES HANDLED:
------------------
✓ 0 nominees for a gender → allow voting for other gender only
✓ 1 nominee → direct winner, still show for voting
✓ 2+ nominees → normal voting, top 2 become winners
✓ Flat already voted → show error immediately
✓ No nominees in wing → error message
✓ Wing deletion with nominees → prevented
✓ Nominee deletion with votes → prevented
